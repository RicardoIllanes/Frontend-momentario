## "el jakeo"
como sabrán, render se "duerme" a los 5 minutos despues de no tener uso, por lo mismmo hice un "jakeo"
el sitio uptimerobot permite hacer consultas cada 5 minutos a un sitio, por lo que en el backend, 
en este caso lo puse en naves, pero ustedes deberán hacer un controller justamente para estando, con el siguiente código

@GetMapping("/health")
public ResponseEntity<String> health() {
    return ResponseEntity.ok("OK");
}

este justamente retorna un ok cada 5 minutos, usando el uptimerobot, lo que si deben agregar esta dependencia al pom
<dependency>
	<groupId>org.springframework.boot</groupId>
	<artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

y en el application.properties
management.endpoints.web.exposure.include=health,info
management.endpoint.health.show-details=always
este de abajo, justamente ya está, pero se tiene que editar con lo que está ahora
spring.autoconfigure.exclude=org.springframework.boot.autoconfigure.security.servlet.SecurityAutoConfiguration,org.springframework.boot.actuate.autoconfigure.security.servlet.ManagementWebSecurityAutoConfiguration

con este pequeño cambio, se puede tener el backend andando siempre sin necesidad de pagar los 7 dólares

##subir imagenes 
para esto vamos a usar imagebb https://imgbb.com/ se crean una cuenta y luego van a este enlace 
https://api.imgbb.com/ que nos ayudará a generar nuestro key para poder enviar imágenes a esta api, este valor lo podemos dejar plano
en nuestro proyecto o para agregar una capa de protección vamos a guardarlo como variable de entorno en vercel, para hacerlo vamos a ir a 
nuestro proyecto y en setting, seleccionamos Environments y ahí Add Environment Variable, en mi caso la agregué como VITE_IMGBB_API_KEY que es el 
valor que verán en el proyecto, recordar que para subir imagenes se usa la url

https://api.imgbb.com/1/upload

ahora vamos a añadir una dependencia a nuestro proyecto que se llama react-image-file-resizer, para añadirla vamos a escribir en la terminal

npm install react-image-file-resizer 

(dentro de la carpeta del proyecto)

generé un archivo llamado UploadImage.js dentro de utils, ahí está todo el tejemaneje de como se usa, el create modal, se actualizó
para ahora aceptar input de imagen, igual que se creó el átomo InputFile que nos servirá para lo que quieran.

ahora el proyecto hace lo siguiente:

- comprime la imagen y al transforma a webp
- la envía con un post a imgbb y recibe la ruta de esta
- esta ruta la guarda en la base de datos

esto hace que nuestro proyecto no dependa de espacio en el servidor ni nada, optimizando recursos
